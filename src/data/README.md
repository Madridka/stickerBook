# Игровые данные

Геометрия альбома каждой сборной хранится рядом с данными игроков в `album.json`. Файл задаёт размеры страниц, PNG-фон, позиции и ширину слотов. Высота слота вычисляется из общего соотношения карточки и в JSON не дублируется.

Коллекции хранятся в `src/data/`. Для каждой коллекции используется отдельная папка сезона, например `src/data/wc-26/`, а внутри неё — папка сборной в нижнем регистре: `src/data/wc-26/mexico/`.

Чтобы добавить новую сборную, создайте её папку внутри нужной коллекции и добавьте в неё файл `players.json` с массивом карточек игроков. Идентификаторы сборной должны быть на английском языке и оставаться стабильными.

Поля `players.json`:

- `id` — уникальный идентификатор карточки;
- `number` — номер игрока;
- `firstName`, `lastName`, `fullName` — имя и фамилия;
- `image` — имя файла изображения карточки;
- `position` — позиция (`GK`, `DF`, `MF` или `FW`);
- `team` — название сборной;
- `weight` — вес карточки для игровой механики.

Редкости, золотые версии, варианты карточек, качество, события и сезоны пока не являются частью данных игроков.
